Introduction

A quick how-to and why.

The API of Ibis.js is by-design simple. It aims to provide only the functionality to deliver its features - and no more!
Every map requires an instance of map 'engine' to draw it. And that map 'engine' is then loaded with user-defined layers of GeoJson vector data.
Each layer in its own turn, can itself be configured to use it's own datasource and have it's own display characteristics e.g. color.

And that conceptually, is all there is to know. Now on to the tutorials below for the coding specifics.
